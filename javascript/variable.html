üì¶ Variable JavaScript

Variable adalah wadah atau tempat penyimpanan untuk menyimpan data dalam program JavaScript. Dengan variable, kita dapat menyimpan nilai dan menggunakannya kembali di seluruh program. JavaScript menyediakan tiga cara untuk mendeklarasikan variable: var, let, dan const. Mari kita pelajari perbedaan dan penggunaannya!

üí°

Apa itu Variable?

üìù Konsep Dasar Variable

Variable adalah nama simbolik yang kita berikan untuk sebuah lokasi di memori komputer yang menyimpan nilai. Bayangkan variable seperti kotak penyimpanan dengan label di luarnya. Label adalah nama variable, dan isi kotaknya adalah nilai yang tersimpan. Kita bisa mengubah isi kotak tersebut kapan saja (kecuali untuk const yang nilainya tetap).

üí° Analogi Sederhana
// Bayangkan variable seperti kotak penyimpanan let namaSiswa = "Budi"; // Kotak berlabel "namaSiswa" berisi "Budi" let umur = 17; // Kotak berlabel "umur" berisi angka 17 let sudahLulus = true; // Kotak berlabel "sudahLulus" berisi nilai true // Kita bisa mengubah isi kotak umur = 18; // Sekarang kotak "umur" berisi 18

1. Gunakan nama yang deskriptif: let totalHarga lebih baik dari let x
2. camelCase untuk variable: let namaLengkap bukan let nama_lengkap
3. UPPERCASE untuk konstanta: const MAX_SIZE = 100
4. Hindari nama singkat: Gunakan let jumlahSiswa bukan let js
5. Konsisten: Pilih satu gaya dan gunakan di seluruh kode

üîë

Tiga Cara Deklarasi Variable

üî¥
var
var

Cara lama untuk mendeklarasikan variable. Memiliki function scope dan dapat di-redeclare. Sudah jarang digunakan dalam kode modern karena perilakunya yang kurang predictable.

  • Function scoped (tidak block scoped)
  • Bisa di-redeclare (deklarasi ulang)
  • Bisa di-reassign (ubah nilai)
  • Hoisting (diangkat ke atas)
  • Tidak direkomendasikan lagi
var nama = "Budi"; console.log(nama); // "Budi" // Bisa di-redeclare var nama = "Siti"; console.log(nama); // "Siti" // Tidak block scoped if (true) { var umur = 25; } console.log(umur); // 25 (masih bisa diakses)
üü¢
let
let

Cara modern untuk mendeklarasikan variable yang nilainya bisa berubah. Memiliki block scope dan tidak bisa di-redeclare. Ini adalah pilihan terbaik untuk variable yang nilainya akan diubah.

  • Block scoped (hanya di dalam {})
  • Tidak bisa di-redeclare
  • Bisa di-reassign (ubah nilai)
  • Tidak ada hoisting yang membingungkan
  • Direkomendasikan untuk nilai berubah
let nama = "Budi"; console.log(nama); // "Budi" // Bisa di-reassign nama = "Siti"; console.log(nama); // "Siti" // Block scoped if (true) { let umur = 25; console.log(umur); // 25 } // console.log(umur); // Error!
üîµ
const
const

Digunakan untuk mendeklarasikan konstanta - variable yang nilainya tidak akan berubah. Harus diinisialisasi saat deklarasi dan tidak bisa di-reassign. Pilihan terbaik untuk nilai yang tetap.

  • Block scoped (hanya di dalam {})
  • Tidak bisa di-redeclare
  • Tidak bisa di-reassign
  • Harus diinisialisasi saat deklarasi
  • Direkomendasikan untuk nilai tetap
const PI = 3.14159; console.log(PI); // 3.14159 // Tidak bisa di-reassign // PI = 3.14; // Error! // Harus diinisialisasi // const nama; // Error! // Object/array bisa dimodifikasi const siswa = { nama: "Budi" }; siswa.nama = "Siti"; // OK ‚úì

üîç Contoh Perbandingan Scope
// ===== VAR: Function Scope ===== function testVar() { var x = 1; if (true) { var x = 2; // Variable yang sama! console.log(x); // 2 } console.log(x); // 2 (terpengaruh!) } // ===== LET: Block Scope ===== function testLet() { let x = 1; if (true) { let x = 2; // Variable berbeda! console.log(x); // 2 } console.log(x); // 1 (tidak terpengaruh!) } // ===== CONST: Block Scope + Immutable ===== function testConst() { const x = 1; if (true) { const x = 2; console.log(x); // 2 } console.log(x); // 1 // x = 3; // Error! Tidak bisa reassign }
‚ö†Ô∏è Kesalahan Umum dengan const

const tidak membuat nilai immutable untuk object/array! const hanya mencegah reassignment variable, bukan perubahan properti object atau elemen array.

const arr = [1, 2, 3];
arr.push(4); // OK ‚úì (mengubah isi array)
arr = [5, 6]; // Error! ‚úó (reassign variable)

Untuk membuat object benar-benar immutable, gunakan Object.freeze().

üéØ

Scope Variable

üí° Apa itu Scope?

Scope menentukan di mana variable dapat diakses dalam kode. Ada tiga jenis scope utama: Global Scope (bisa diakses di mana saja), Function Scope (hanya di dalam fungsi), dan Block Scope (hanya di dalam blok kode {}). Memahami scope sangat penting untuk menghindari bug dan menulis kode yang bersih.

üåç 1. Global Scope
// Variable di luar fungsi/blok = Global Scope let namaSekolah = "SMA Negeri 1"; function tampilkanSekolah() { console.log(namaSekolah); // Bisa diakses ‚úì } if (true) { console.log(namaSekolah); // Bisa diakses ‚úì } tampilkanSekolah(); // "SMA Negeri 1"
‚öôÔ∏è 2. Function Scope (var)
function hitungTotal() { var total = 100; // Function scoped if (true) { var total = 200; // Masih variable yang sama } console.log(total); // 200 } // console.log(total); // Error! Di luar fungsi
üì¶ 3. Block Scope (let & const)
// Block scope: hanya di dalam {} if (true) { let x = 10; const y = 20; console.log(x, y); // 10 20 } // console.log(x); // Error! Di luar blok // console.log(y); // Error! Di luar blok // Loop juga punya block scope for (let i = 0; i < 3; i++) { console.log(i); // 0 1 2 } // console.log(i); // Error! i hanya ada di loop
‚úÖ Tips Menggunakan Scope

1. Gunakan scope terkecil: Deklarasikan variable di scope terkecil yang diperlukan
2. Hindari global variable: Terlalu banyak global variable dapat menyebabkan konflik
3. Gunakan let/const di loop: for (let i = 0; ...) bukan for (var i = 0; ...)
4. Block scope lebih aman: let dan const mencegah bug yang tidak terduga
5. Nama variable unik per scope: Hindari nama variable yang sama di scope berbeda

Hoisting

üí° Apa itu Hoisting?

Hoisting adalah mekanisme JavaScript yang "memindahkan" deklarasi variable dan fungsi ke bagian atas scope mereka sebelum kode dieksekusi. Namun, hanya deklarasinya yang dipindahkan, bukan inisialisasinya. Ini bisa menyebabkan perilaku yang membingungkan, terutama dengan var.

üî¥ Hoisting dengan var
// Kode yang kita tulis: console.log(nama); // undefined (bukan error!) var nama = "Budi"; console.log(nama); // "Budi" // Yang sebenarnya terjadi (hoisting): var nama; // Deklarasi dipindahkan ke atas console.log(nama); // undefined nama = "Budi"; // Inisialisasi tetap di sini console.log(nama); // "Budi"
üü¢ Temporal Dead Zone (let & const)
// let dan const juga di-hoist, tapi tidak bisa diakses // console.log(nama); // Error! Temporal Dead Zone let nama = "Budi"; console.log(nama); // "Budi" // Temporal Dead Zone: area sebelum deklarasi // Di zona ini, variable sudah ada tapi belum bisa diakses function contoh() { // TDZ mulai di sini // console.log(x); // Error! let x = 10; // TDZ berakhir di sini console.log(x); // 10 }
‚ö†Ô∏è Mengapa Hoisting Bisa Berbahaya

Hoisting dengan var bisa menyebabkan bug yang sulit dilacak karena:

1. Nilai undefined yang tak terduga: Variable yang digunakan sebelum inisialisasi bernilai undefined
2. Bug tersembunyi: Kode tidak error tapi hasilnya salah
3. Sulit di-debug: Perilaku tidak intuitif dan membingungkan

Solusi: Gunakan let dan const yang memberikan error jelas ketika diakses sebelum deklarasi!

üéì

Kapan Menggunakan var, let, atau const?

‚úÖ Best Practice Modern JavaScript

1. Default ke const: Gunakan const sebagai pilihan utama
2. let untuk nilai berubah: Hanya gunakan let jika nilai akan berubah
3. Jangan gunakan var: var sudah deprecated dalam kode modern
4. Deklarasi di awal scope: Deklarasikan variable di awal untuk kejelasan
5. Satu deklarasi per baris: Hindari let a = 1, b = 2;

üîµ Gunakan const untuk:
// 1. Nilai tetap / konstanta const PI = 3.14159; const MAX_SISWA = 30; const NAMA_SEKOLAH = "SMA Negeri 1"; // 2. Object/Array yang propertinya berubah const siswa = { nama: "Budi", umur: 17 }; siswa.umur = 18; // OK ‚úì // 3. Fungsi yang tidak akan di-reassign const hitungLuas = (p, l) => p * l; // 4. Import module const express = require('express');
üü¢ Gunakan let untuk:
// 1. Counter dalam loop for (let i = 0; i < 5; i++) { console.log(i); } // 2. Variable yang nilainya berubah let total = 0; total += 10; total += 20; // 3. Variable dalam kondisi let status; if (nilai >= 75) { status = "Lulus"; } else { status = "Tidak Lulus"; } // 4. Akumulator let hasil = ""; for (let char of "Hello") { hasil += char + " "; }
üî¥ Jangan gunakan var:
// ‚ùå JANGAN LAKUKAN INI var nama = "Budi"; var umur = 17; // ‚úÖ GUNAKAN INI const nama = "Budi"; let umur = 17; // Alasan: var punya masalah hoisting, function scope, // dan bisa di-redeclare yang menyebabkan bug
üí° Aturan Emas

Default ke const, upgrade ke let jika perlu, jangan pernah gunakan var.

Mulailah dengan mendeklarasikan semua variable menggunakan const. Jika kemudian Anda menemukan bahwa nilai variable perlu berubah, ubah const menjadi let. Pendekatan ini membuat kode Anda lebih aman dan mudah diprediksi.
