ğŸ¯ Mengapa Perlu Return?

Return digunakan untuk mengembalikan hasil dari function sehingga bisa digunakan di bagian kode lain. Tanpa return, function hanya bisa menampilkan output tapi tidak bisa menyimpan hasilnya.

âŒ Tanpa Return:

    console.log(a * b);
}
kaliBilangan(3, 4);
// Hanya tampil, tidak bisa disimpan!

Hasilnya tidak bisa digunakan ulang

âœ… Dengan Return:

function kaliBilangan(a, b) {
    return a * b;
}
let hasil = kaliBilangan(3, 4);
console.log(hasil);
// 12

Hasil bisa disimpan dan digunakan ulang!

// Struktur Dasar Return
function namaFunction() {
   // Kode
    return "Nilai yang dikembalikan";
}

console.log(hasil);

Arrow Function adalah cara penulisan function yang lebih singkat dan modern (ES6+). Menggunakan simbol => (panah) untuk merepresentasikan function. Sama saja hasilnya, tapi penulisannya lebih elegant! âœ¨

âš¡ Arrow Function:

// Berbagai bentuk Arrow Function:
// 1. Dengan satu parameter (bisa tanpa kurung)
const kuadrat = x => x * x;
// 2. Dengan dua parameter
const kali = (a, b) => a * b;
// 3. Implicit return (satu baris)
const sapa = nama => `Halo ${nama}!`;
// 4. Explicit return (lebih dari satu baris)

    let hasil = data * 2;
    returnhasil;
};

ğŸ“ Apa itu Scope?

Scope adalah area atau "jangkauan" di mana sebuah variabel bisa diakses. Ada 3 jenis scope: Global, Function, dan Block scope.

ğŸŒ Global Scope

Variabel bisa diakses di mana saja dalam kode

    console.log(nama);
}

ğŸ¯ Function Scope

Variabel hanya bisa diakses dalam function tertentu

    let nama = "Budi";
    console.log(nama);
}

ğŸ“¦ Block Scope

    let x = 5;
    console.log(x);

// x tidak bisa diakses di sini
// ========== GLOBAL SCOPE ==========

function fungsiA() {
   // ========== FUNCTION SCOPE ==========
    let nilaiFungsi = 50;
    console.log(nilaiGlobal);
   // âœ… Bisa (karena global)
    console.log(nilaiFungsi);
   // âœ… Bisa (function scope)
}
fungsiA();
console.log(nilaiGlobal);
// âœ… Bisa (karena global)
console.log(nilaiFungsi);
// âŒ Error! (karena function scope)
